iOS错误积累

1.receiving collection element of type int is not an objective c object in iOS

语法糖字典里value要是对象

2.writetofile fail

路径要存在文件夹 光有路径无文件夹写入失败

3.- (BOOL)copyItemAtPath:(NSString )srcPath toPath:(NSString )dstPath error:(NSError **)error

新路径下要有一个相同名字的文件 否则不成功

4.Message from debugger: Terminated due to Memory Error

maybe内存分配的时候，突然间占用太多，太快，OS给你关掉了app。

5.Analyze提示:Value stored to “xxx”is never read

变量没有被使用

6.删除cell uncaught exception ‘NSInternalInconsistencyException’, reason: ‘Invalid update: invalid number of rows in section 0. The number of rows contained in an existing section after the update (0) must be equal to the number of rows contained in that section before the update (3), plus or minus the number of rows inserted or deleted from that section (0 inserted, 1 deleted) and plus or minus the number of rows moved into or out of that section (0 moved in, 0 moved out).’

原因：deleteRowsAtIndexPaths后Section里面的rows数量，和系统调用numberOfRowsInSection时rows数量不一致
解决：先删掉数据 再删cell
