发表于 2015-07-20   |   分类于 iOS轮子   |  

Mantle makes it easy to write a simple model layer for your Cocoa or Cocoa Touch application.Mantle

为什么唱吧iOS 6.0选择了Mantle

2.0 CHANGELOG

Explicit JSON key paths 明确JSON路径

+JSONKeyPathsForPropertyKey 这里面必须写每一个属性映射，忽略任何属性将不会被考虑JSON序列化和反序列化。
可以使用+[NSDictionary mtl_identityPropertyMapWithModel：] 自动创建一个一对一的映射相匹配的以前的默认行为。

Predefined transformers now part of JSON adapter 预定义的transformers现在转移到了MTLJSONAdapter

+mtl_JSONDictionaryTransformerWithModelClass: 和 +mtl_JSONArrayWithModelClass:转移到了MTLJSONAdapter

Core Data adapter now separate

MTLManagedObjectAdapter是一个适配器转换 MTLModel实例和 Core Data managed对象。
MTLManagedObjectAdapter类被移到了自己的框架

Managed object transformers reversed

In addition to being a separate framework, the behavior of MTLManagedObjectAdapter has changed as well—specifically, the direction of managed object attribute transformers has been flipped.
Managed object transformers now convert from managed object attributes to model properties in the forward direction. In the reverse direction, they convert from properties to managed object attributes.
To update:
Swap the forward and reverse transformation logic of any custom managed object transformers, or use -mtl_invertedTransformer to do it automatically. 嗯 没看懂

OS X 10.9 and iOS 8

最低要求

JSON key paths can only traverse objects

+JSONKeyPathsByPropertyKey 必须是一个对象（字典）
以前可以使用数组，现在不行了
如果你使用一个数组作为一个关键路径的元素更新您的JSON变压器 。
